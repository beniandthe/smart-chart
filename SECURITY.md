# Security Policy

## Supported Versions

Security fixes are applied to `main` and then carried forward into active feature branches.

## Reporting A Vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting or contact the repository owner directly with:

- affected area or workflow
- reproduction steps
- expected impact
- any screenshots, logs, or sample files needed to understand the issue

We will acknowledge valid reports as quickly as possible and prioritize fixes based on severity, exploitability, and user impact.

## Automation

This repository uses GitHub Actions, CodeQL, Dependabot, and dependency review to catch regressions and dependency/security issues early. These checks are guardrails, not a replacement for careful review.
